# 관리자 페이지 모바일 레이아웃 개선 방안

## 현재 문제
- 좌측 채팅 목록(300px)이 화면을 너무 많이 차지
- 모바일에서 채팅 영역이 좁아서 사용성 저하

---

## 방안 1: 반응형 화면 전환 (추천 ⭐)

**설명:** 화면 크기에 따라 완전히 다른 레이아웃 사용

| 화면 | 레이아웃 |
|------|----------|
| 데스크톱 (>768px) | 현재처럼 좌우 분할 |
| 모바일 (≤768px) | 목록/채팅 화면 전환 |

**모바일 동작:**
1. 처음: 채팅 목록만 표시 (전체 화면)
2. 채팅방 선택 → 채팅 화면으로 전환
3. 뒤로가기 버튼 → 목록으로 복귀

**장점:**
- 각 화면에서 최대 공간 활용
- 직관적인 UX (카카오톡, 메신저 앱과 동일)
- 구현 난이도 중간

**단점:**
- 화면 전환 필요

---

## 방안 2: 슬라이드 드로어 (Drawer)

**설명:** 채팅 목록을 왼쪽에서 슬라이드로 열고 닫음

**동작:**
1. 기본: 채팅 화면만 표시
2. 햄버거 메뉴(☰) 클릭 → 좌측에서 목록 슬라이드
3. 채팅방 선택 → 드로어 자동 닫힘

**장점:**
- 채팅 영역 최대화
- 익숙한 패턴 (앱 사이드메뉴)

**단점:**
- 목록 확인하려면 매번 드로어 열어야 함
- 새 메시지 알림 확인 어려움

---

## 방안 3: 바텀시트 (Bottom Sheet)

**설명:** 하단에서 채팅 목록이 올라옴

**동작:**
1. 기본: 채팅 화면 + 하단에 미니 바
2. 하단 바 위로 스와이프 → 목록 표시
3. 채팅방 선택 → 시트 닫힘

**장점:**
- 한 손 조작 용이
- 현재 채팅 유지하면서 목록 확인 가능

**단점:**
- 구현 복잡도 높음
- 익숙하지 않은 패턴

---

## 방안 4: 좌측 패널 축소 모드

**설명:** 좌측 패널을 아이콘만 보이는 축소 모드로 전환

| 모드 | 좌측 패널 |
|------|-----------|
| 축소 | 60px (아이콘+배지만) |
| 확장 | 300px (전체 목록) |

**동작:**
1. 모바일: 기본 축소 모드
2. 토글 버튼으로 확장/축소
3. 채팅방 선택 시 자동 축소

**장점:**
- 항상 목록 접근 가능
- 새 메시지 배지 확인 가능

**단점:**
- 축소 시 정보 제한적
- 확장/축소 전환 필요

---

## 방안 5: 상단 탭 네비게이션

**설명:** 상단에 탭으로 목록/채팅 전환

```
[채팅 목록] [현재 채팅]
```

**동작:**
1. "채팅 목록" 탭: 전체 목록 표시
2. "현재 채팅" 탭: 선택된 채팅 표시
3. 목록에서 선택 시 자동으로 채팅 탭 전환

**장점:**
- 간단한 구현
- 명확한 화면 구분

**단점:**
- 탭 영역이 공간 차지
- 빠른 전환 시 불편

---

## 방안 6: 스와이프 제스처

**설명:** 좌우 스와이프로 화면 전환

**동작:**
1. 왼쪽 스와이프 → 채팅 목록
2. 오른쪽 스와이프 → 채팅 화면

**장점:**
- 제스처로 빠른 전환
- UI 깔끔 (버튼 없음)

**단점:**
- 발견성 낮음 (처음 사용자 모를 수 있음)
- 메시지 스크롤과 충돌 가능성
